# LangGraph 学习路线与复习索引

这套整理版按“先会搭图，再会控制执行，再补生产能力和实战”的顺序重组。原始 Notion 导出的文件保留在 `LangGraph笔记` 目录；整理版图片统一放在 `figs/` 并使用相对路径引用。

## 推荐阅读顺序

1. [01.LangGraph概览与基础组件](01.LangGraph概览与基础组件)：理解 LangGraph 的定位、五大优势、State/Node/Edge 和最小图结构。
2. [02.图编排与控制流](02.图编排与控制流)：掌握串行、分支、条件边、循环、动态模型切换和 map-reduce。
3. [03.持久化与记忆](03.持久化与记忆)：理解 checkpointer、thread_id、跨线程 Store、短期记忆、长期记忆、消息过滤和总结。
4. [04.人机交互与状态编辑](04.人机交互与状态编辑)：学习 interrupt、等待用户输入、工具调用审查和图状态编辑。
5. [05.时光旅行与流式输出](05.时光旅行与流式输出)：学习重放、分叉，以及 values/updates/debug/messages 四类 stream_mode。
6. [06.工具调用](06.工具调用)：掌握工具定义、绑定、模型工具调用和执行工具结果。
7. [07.Subgraph与多智能体](07.Subgraph与多智能体)：学习主图/子图拆分和多智能体协作模式。
8. [08.实战案例与生产补充](08.实战案例与生产补充)：整理代码分析助手案例，并保留生产环境后续补充位置。

## 复习方法

- 第一遍：只读每篇开头的“复习抓手”和流程图，先建立整体地图。
- 第二遍：按 `01 -> 04 -> 06` 跑通一个能中断、能调用工具的最小 Agent 图。
- 第三遍：补 `03` 的持久化与记忆，再用 `05` 练习重放、分叉和流式调试。
- 项目复盘：重点追问每个节点的输入/输出 State、每条边的触发条件、是否需要 checkpointer、是否需要人工介入。

## 大节划分原则

- 基础组件负责“图由什么组成”。
- 控制流负责“图怎么走”。
- 持久化与记忆负责“图如何跨轮次、跨线程保存上下文”。
- 人机交互和时间旅行负责“运行中如何暂停、审查、回放和改写”。
- 工具调用和多智能体负责“Agent 如何行动，以及多个图/智能体如何协作”。
## 使用建议补充

这组笔记适合按“能力递进”复习，而不是按 API 名称死记。LangGraph 的核心不是多了一个画图工具，而是把 Agent 的执行过程显式建模成“状态 + 节点 + 边 + 检查点”的工作流。

| 复习阶段 | 目标 | 重点问题 |
| --- | --- | --- |
| 入门 | 能跑通最小图 | State 怎么定义，Node 返回什么，Edge 怎么连接 |
| 编排 | 能表达复杂流程 | 什么时候用条件边，什么时候用循环，什么时候拆成子图 |
| 可靠性 | 能恢复和审查 | checkpointer、thread_id、interrupt 分别解决什么问题 |
| Agent 化 | 能接工具和记忆 | 工具结果如何回写，短期记忆和长期记忆如何分层 |
| 工程化 | 能调试和上线 | 如何 stream、如何 replay、如何记录执行轨迹 |

复习时建议把每一篇都还原成一句话：这一篇解决“图执行”的哪类问题。只要能说清楚问题、机制和适用场景，后续 API 细节就容易查漏补缺。
