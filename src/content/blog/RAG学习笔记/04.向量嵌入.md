# 向量嵌入

向量嵌入就是把文本、图像、音频等复杂数据转换成固定长度的数值向量，让计算机可以用数学方式比较它们之间的**语义相似度**。文档指出，Embedding（向量嵌入）会把复杂、高维的数据对象转换成低维、稠密、连续的数值向量，例如文本、图片等都可以被转换成一个固定长度的一维数组。

Embedding（向量嵌入）的**核心思想**是：

> **语义相似的内容，在向量空间中距离更近；语义无关的内容，在向量空间中距离更远**。
> 

# 如何衡量两个向量是否相似

| 方法 | 中文解释 | 含义 | 特点 |
| --- | --- | --- | --- |
| Cosine Similarity（余弦相似度） | 看两个向量方向是否一致 | 值越接近 1，语义越相似 | 最常用 |
| Dot Product（点积） | 计算两个向量对应位置乘积之和 | 归一化后等价于余弦相似度 | 计算简单 |
| Euclidean Distance（欧氏距离） | 计算两个向量之间的直线距离 | 距离越小，语义越相似 | 直观但受向量尺度影响 |

# Embedding（向量嵌入）在 RAG 中的作用

**语义检索的基础**

RAG 的“检索”环节通常以基于 Embedding 的语义搜索为核心。 

（1）**离线索引构建**：将知识库内文档切分后，使用 Embedding 模型将每个文档块（Chunk）转换为向量，存入专门的向量数据库中。

（2）**在线查询检索**：当用户提出问题时，使用**同一个** Embedding 模型将用户的问题也转换为一个向量。

（3）**相似度计算**：在向量数据库中，计算“问题向量”与所有“文档块向量”的相似度。

（4）**召回上下文**：选取相似度最高的 Top-K 个文档块，作为补充的上下文信息，与原始问题一同送给大语言模型（LLM）生成最终答案。

**决定检索质量的关键**

Embedding 的质量直接决定了 RAG 检索召回内容的准确性与相关性。一个优秀的 Embedding 模型能够精准捕捉问题和文档之间的深层语义联系，即使用户的提问和原文的表述不完全一致。反之，一个劣质的 Embedding 模型可能会因为无法理解语义而召回不相关或错误的信息，从而“污染”提供给 LLM 的上下文，导致最终生成的答案质量低下。

# RAG 对 Embedding 提出了新要求

| 新要求 | 含义 |
| --- | --- |
| 领域自适应能力 | 法律、医疗、科研等专业领域需要理解专业术语 |
| 多粒度与多模态支持 | 不只处理短文本，还要处理长文档、代码、图像、表格等 |
| 检索效率与混合检索 | 既要效果好，也要速度快、存储成本低，还可能结合稠密检索和关键词检索 |

# 嵌入模型是怎么训练出来的

## MLM（掩码语言模型）

```python
原句：我今天学习 RAG 知识。
处理后：我今天学习 [MASK] 知识。
目标：让模型预测 [MASK] = RAG
```

## NSP（下一句预测）

```python
句子 A：RAG 需要先进行文档检索。
句子 B：然后把检索结果交给大语言模型生成答案。
判断：B 是否是 A 的下一句？
```

## 度量学习和对比学习

为了让 Embedding（向量嵌入）更适合检索任务，现代嵌入模型通常还会使用 Metric Learning（度量学习）和 Contrastive Learning（对比学习）。文档指出，度量学习的目标是让正例文本对在向量空间中更近，让负例文本对更远；对比学习通常构造 Anchor（锚点）、Positive（正样本）、Negative（负样本）三元组，优化正负样本之间的距离关系。

```python
问题：什么是 RAG？
正样本：RAG 是检索增强生成。
负样本：YOLOv5 是目标检测算法。
```
