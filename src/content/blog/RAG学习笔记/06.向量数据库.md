# 向量数据库

如何从海量高维向量中，快速找到和用户查询最相似的内容？

# 向量数据库的主要作用

| 功能 | 解释 |
| --- | --- |
| 高效相似性搜索 | 使用 HNSW（分层可导航小世界图）、IVF（倒排文件索引）等索引技术，在海量向量中快速找到相似内容 |
| 高维数据存储与管理 | 专门存储成百上千维的向量，并支持增、删、改、查 |
| 丰富查询能力 | 不仅能做向量相似度检索，还可以结合元数据过滤，例如“年份 > 2023” |
| 可扩展与高可用 | 支持分布式架构，可以通过增加节点处理更大规模的数据 |
| AI（人工智能）生态集成 | 可以和 LangChain（大模型应用开发框架）、LlamaIndex（大模型数据框架）等工具结合使用 |

# 向量数据库的工作原理

| 层级 | 作用 |
| --- | --- |
| 存储层 | 存储向量数据和元数据，支持分布式存储 |
| 索引层 | 创建和维护 HNSW、LSH、PQ 等索引 |
| 查询层 | 处理用户查询，支持相似性搜索和混合查询 |
| 服务层 | 管理客户端连接、监控、日志和安全管理 |

其中，最关键的是 **索引层**。因为向量数据库能不能快速检索，主要取决于索引算法的设计

# 几类向量检索技术

## 1. 基于树的方法

代表方法是 Annoy（Approximate Nearest Neighbors Oh Yeah，近似最近邻搜索库）。它使用随机投影树，把向量空间不断划分，从而减少搜索范围。适合做快速近似搜索。

## 2. 基于哈希的方法

代表方法是 LSH（Locality Sensitive Hashing，局部敏感哈希）。它的核心思想是：**相似的向量经过哈希函数后，更有可能被分到同一个桶里**。这样查询时就不需要扫描所有向量，只需要在相关桶中搜索。

## 3. 基于图的方法

代表方法是 HNSW（Hierarchical Navigable Small World，分层可导航小世界图）。它通过多层邻近图来加速搜索。可以理解为：系统先在高层图中快速定位大概区域，再逐步进入底层图中精细查找。HNSW 是当前向量数据库中非常常见的索引方法。

## 4. 基于量化的方法

代表方法包括 FAISS（Facebook AI Similarity Search，Facebook 人工智能相似性搜索库）中的 IVF（Inverted File，倒排文件索引）和 PQ（Product Quantization，乘积量化）。这类方法通过聚类和压缩向量来减少存储和计算成本。
